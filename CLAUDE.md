# CLAUDE.md — Project Instructions for Claude Code

## Project
gh-setup — A Go CLI that interactively configures GitHub organizations/individuals, repositories, branch protection, teams, labels, CI, and governance from a YAML config file. Distributable as a `gh` CLI extension.

## Git Rules
- NEVER add "Co-Authored-By", "Generated by Claude", or any AI attribution lines in commit messages, PR descriptions, merge commits, or any git-related text.
- Keep commit messages concise and conventional (e.g., `feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`).

## Tech Stack
- Language: Go
- CLI: cobra
- Interactive prompts: charmbracelet/huh + charmbracelet/lipgloss
- Logging: charmbracelet/log
- Config: gopkg.in/yaml.v3
- GitHub API: google/go-github/v68
- Auth: golang.org/x/oauth2
- Templates: go:embed

## Architecture
See `specs.md` for full spec. Key layout:
- `main.go` — entry point
- `cmd/` — cobra commands (root, init, apply, diff)
- `internal/config/` — YAML config structs
- `internal/wizard/` — interactive init wizard (huh/bubbletea)
- `internal/github/` — GitHub API modules (org, repo, protection, labels, teams, security, secrets)
- `internal/templates/` — CI workflow and governance file templates (go:embed)
- `internal/apply/` — idempotent apply logic
- `internal/diff/` — config vs actual state comparison
